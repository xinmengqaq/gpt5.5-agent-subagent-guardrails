# 子代理轻量日志

用途：缓解子代理执行黑盒，给主代理提供低 token 的执行索引。日志不是验收证据本体，不能代替测试、浏览器截图、命令输出或审查结论。

## 写入位置

- 在当前项目根目录创建或使用 `subagent-log/`。
- 每天一个日志文件，按项目本地日期命名：`subagent-log/YYYY-MM-DD-subagent-log.md`。
- 如果执行跨天，写入新日期文件；不要继续写昨天的文件。
- 并行子代理追加日志前必须重新读取当天文件，只追加自己的段落，不覆盖他人内容；发生写入冲突时重新读取后再追加。

## 写入时机

- 每个子代理在完成、返工、阻塞或被要求复查时追加一段。
- 返工子代理追加新段落，不改写旧段落。
- 主代理可在派发提示词中要求子代理执行日志写入，但模型选择理由仍只留在主代理内部，不写入日志。

## 单段格式

每段控制在 5-8 行，只写事实索引，不写长篇推理。

```markdown
## HH:mm Task <编号/名称> / Step <编号/名称> / <模型>

- 身份：<前端专家/后端专家/测试与验收专家/代码质量审查专家等>
- 范围：<主要文件、模块或页面>
- 做了：<一句话事实结果>
- 证据：<命令、测试、浏览器路径、截图路径、日志路径或阻塞信息>
- 状态：<合格 / 返工 / 阻塞>
- 风险：<剩余风险；没有则写“无”>
```

## 禁止

- 不写内部推理链、长篇排查过程、模型选择理由或无证据的自评。
- 不把日志当作完成证明；完成仍以真实检查、审查、截图、命令或可观察行为为准。
- 不为了写日志扩大任务范围、修改 Task 边界或补做主代理没有派发的工作。
